* First Window
Prose window — one "* Heading" at the top level = one <section class="window">
in the rendered page. The heading text is the window's title chip.

Ordinary Org markup works: [[https://example.com][links]], *emphasis*, =code=,
lists, block quotes, etc. Images export via org-html.

* A Second Window
Each top-level heading is its own window. Break posts into short windows so
the BBS-style frame works visually — long prose inside one window is fine,
but most posts flow better as two or three stanzas.

* Interactive Demo                                         :experiment:
The =:experiment:= tag on a heading gives that single window the yellow
frame. Drop raw HTML into the window body with an EXPORT block:

#+BEGIN_EXPORT html
<p>Put canvas, buttons, or any HTML widget here. This block is passed
through verbatim — no Org processing happens inside.</p>
#+END_EXPORT

* Tangled assets (optional)
For experiment posts that ship JS or CSS, use Babel =:tangle= blocks.
The first block writes a <link> into post.extra-head.html (which
tools/build.py injects into <head>). The second ships a JS file.

#+BEGIN_SRC html :tangle post.extra-head.html
<link rel="stylesheet" href="/assets/new-post-slug/style.css?v={{ ASSET_VERSION }}">
#+END_SRC

#+BEGIN_SRC html :tangle post.extra-body.html
<script src="/assets/new-post-slug/main.js?v={{ ASSET_VERSION }}"></script>
#+END_SRC

#+BEGIN_SRC js :tangle ../../../site/assets/new-post-slug/main.js :mkdirp yes
// Your experiment code. Tangle writes this verbatim to
// site/assets/new-post-slug/main.js on every export.
console.log("hello from new-post-slug");
#+END_SRC
