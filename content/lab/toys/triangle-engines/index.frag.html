Both buttons trigger the same bell synthesis through the same envelope, added sine partials, and bandpass filter. The only difference is the triangle wave source.

  • Native uses the browser's built-in oscillator.type = 'triangle'.
  • NES-style uses a Web Audio PeriodicWave built by createNesTriangleWave from a 32-step NES-style source table, projected onto 24 cosine/sine harmonics with the DC offset removed.

A 32-step NES-style source table reconstructed as a 24-harmonic PeriodicWave is a different audio artifact from the browser's native triangle oscillator. The contrast is audible when both pass through identical bell synthesis.

The page shows two buttons. Each click triggers a bell tone at a random MIDI note from the same scale. The only thing that changes between the two paths is the triangle wave source passed to strikeBell.

The chewgum-dsp library exists because the NES-style PeriodicWave has a different audible character from the browser's native triangle oscillator. This page is the proof: same envelope, same filter, same scale, only the wave differs.
